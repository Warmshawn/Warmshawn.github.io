U盘安装Ubuntu

Posted on 2019-09-23 Edited on 2019-09-24 In Tech - Q
安装 Widnows 10 + Ubuntu 18 双系统

前期准备

一台电脑, 已经装了Windows. 同时还留有足够大的硬盘空间留给Linux (建议至少32G).
一个ubuntu镜像: ubuntu-18.04.3-desktop-amd64.iso
一个U盘, 已格式化成FAT32格式. 为了安装快速, 建议使用高速U盘.
一个刻录软件, rufus. 轻量级开源免费无广告！不建议用UltraISO, 尽管刻录不会有问题, 但我在U盘启动时会出现这个问题：”Failed to load ldlinux.c32”

在Windows下划出一部分空间给Linux

右键计算机→管理→磁盘管理→右键要分区的磁盘→压缩卷→输入大小；分出的空间不加盘符 ，具体操作如下图

PS：这部分分出的空间在windows下就不可使用了，windows不能访问Ubuntu的磁盘，但在Ubuntu中仍可以访问windows的磁盘。

安装

  1. 制作U盘启动盘
    打开rufus

    Device是要用的U盘。
    Boot selection是镜像文件,通过右侧的SELECT选择要用的镜像文件。
    其他默认设置。
    点击START开始制作, 等待, 完成。

  2. 将启动盘插入待安装电脑, 开机, 按F12进入BIOS界面, 选择U盘启动. 在本例中, 选择启动, 如下图所示。
    没有U盘启动项, 进入SETUP——

  3. 语言有中文选项，这里就先选默认的英文。点击“Install Ubuntu”

  4. 这里先不选更新，点击“Continue”

  5. 因为要安装双系统，所以选择最下面的一项“something else” 然后“Continue”

  6. 点击空闲空间，然后点击“+”号进行分区

    首先是“swap area”，这个分区的空间不大于电脑的物理空间，例如我的电脑运存是4G的，这个分区我就设置为4G，选择逻辑分区“Logical”

  7. 然后是“/”分区，这个分区相当于windows的C盘，这个看个人使用和电脑情况选择，网上有的说一般大于16G就行，任然选择逻辑分区“Logical”。

    /home” 这个分区相当于windows里C盘以外的其他盘，预留出1G的空间，剩余的空间就都可以留给这个分区，任然选择逻辑分区“Logical”。

    “/boot”这个分区是用来启动Ubuntu的，空间就是上一步剩余的1G，注意这个分区要选择主分区“Primary”。之所以把它放到最后，是因为我在试了几次以后发现如果把这一步放在前面的任何一部的话，分到最后就剩下一个分区的空间成为不可用空间，无法进行分配。另外说一下“/boot”分区的空间问题，网上很多教程都说200M够用，但是之后的使用中会经常出现一个问题，就是“The volume “boot” has only 0 byte disk space remaining”很烦人，所以这里建议稍微大一些。

  8. 下面进行到比较关键的一部，就是“启动引导器”的设置。
    启动引导器要设置为前面boot所在的分区,

    网上的教程大部分建议用win7引导Ubuntu，理由是这样比较安全，具体原因我也不清楚大笑

  9. 点击“Install Now”，后面的步骤就是设置用户名和密码按步骤操作之后就能安装了。安装完成后提示立即重启，点击即可。重启之后发现直接进入了win7，那是因为我们还没有配置Ubuntu引导。

  10. 下载EasyBCD安装，进入软件后选择“Add New Entry”，然后按下图设置，Type选择GRUB（Legacy），Drive选择我们的“/boot”分区，Name自定义，然后点击“Add Entry”

    在“View Settings”中就可以看到新加的Ubuntu引导
